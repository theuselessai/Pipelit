## Success Criteria

All of the following must be true before tagging v0.2.0:

### Security
- [ ] No unsandboxed code execution paths remain — `run_command` and `code` without workspace return errors
- [ ] `platform_api` tool `base_url` cannot be overridden by LLM output
- [ ] Redundant tools (`http_request`, `web_search`, `calculator`, `datetime`) fully removed from registry, palette, and type definitions
- [ ] Skill directories properly mounted into bwrap sandbox (PR #116)

### Functionality
- [ ] `human_confirmation` node correctly triggers `interrupt_before` on downstream nodes
- [ ] `aggregator` node fully removed from frontend and node type registry
- [ ] Execution timeouts work correctly — hung processes killed after configurable limit
- [ ] Activity-based timeout watchdog operational (PR #104)
- [ ] Telegram trigger matches by bot token with concurrent polling support (PR #117)
- [ ] `GET /health` returns service status without authentication

### Stability
- [ ] All existing tests pass: `python -m pytest tests/ -v`
- [ ] No regressions in workflow execution (create, run, monitor cycle)
- [ ] WebSocket real-time updates functioning (node status, execution events)
